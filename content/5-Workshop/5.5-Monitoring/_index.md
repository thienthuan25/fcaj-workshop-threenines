#### Overview

In the previous sections, the system was configured to automatically collect, analyze, and send cost alerts. However, we also need a monitoring layer to ensure that the system itself continues to operate reliably. If a Lambda function fails without anyone noticing, the system will no longer function correctly.

In this section, we will configure CloudWatch Alarms to monitor the health of the system. There are two types of incidents that should trigger alerts:

- When a Lambda function encounters an error during execution.
- When a failed processing event is sent to the **Dead Letter Queue**.

These alarms will publish notifications through the same SNS topic that was created earlier, allowing you to receive email alerts immediately whenever the system encounters an issue.

{{% notice info %}}
Note: The system includes two different types of alerts:

- Cost alerts generated by the Analyzer notify you when costs exceed the defined threshold.
- System alerts generated by CloudWatch Alarms notify you when the system itself encounters an error.
{{% /notice %}}
